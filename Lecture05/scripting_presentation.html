El problema

El notebook lo tiene todo mezclado

carga de datos transformaciones definición del modelo loop de entrenamiento evaluación hiperparámetros hardcodeados paths hardcodeados visualización inferencia

Funciona para explorar. Se rompe cuando hay que reutilizar, escalar o trabajar en equipo.

02 / 09
El principio central
Los scripts son dumb.
El src/ es smart.

Toda la lógica vive en src/. Los scripts solo parsean argumentos, cargan config, y llaman a src/.

04 / 09
División de responsabilidades
src/ contiene
  • Carga de datos y transforms
  • Definición del modelo
  • Loops de train/eval
  • Callbacks (checkpoint, early stop)
  • Predictor para inferencia
  • Utilidades de I/O y logging
src/ NUNCA tiene
  • argparse
  • yaml.safe_load
  • Paths hardcodeados
  • print("Training...")
  • Lógica de CLI
  • Conocimiento de argv
05 / 09
El patrón de config

YAML como fuente de verdad

# config/base.yaml gana siempre,
# salvo que el CLI sobreescriba explícitamente

with open(args.config) as f:
    cfg = yaml.safe_load(f)

if args.batch_size:
    cfg["training"]["batch_size"] = args.batch_size

if args.run_name:
    cfg["logging"]["run_name"] = args.run_name

Default: make train  ·  Experimento rápido: --batch-size 128  ·  Sin editar archivos.

06 / 09
Por qué pip install -e .

Python necesita saber
dónde está src/

# Sin instalar → NameError al importar src/
from src.data.dataset import build_dataloaders  ✗

# Con pip install -e .
# Python lee src/ directamente desde tu carpeta
from src.data.dataset import build_dataloaders  ✓

# El -e significa editable:
# cualquier cambio en src/ se refleja sin reinstalar

Reproducibilidad

¿Qué se guarda
del preprocesamiento?

Qué ¿Se guarda? Cómo garantizar consistencia
Pesos del modelo Sí Cargar desde el mismo checkpoint
Tokenizador (NLP) Sí save_pretrained / push_to_hub
ImageNet transforms (CV) No Siempre llamar weights.transforms()
Limpieza de texto custom No Copiar la función en predictor.py
08 / 09
El Makefile

Un menú de comandos
para el proyecto

# Flujo completo
make data
make train
make evaluate  CHECKPOINT=runs/best.pt
make serve     CHECKPOINT=runs/best.pt  PORT=8081

# Smoke test antes de un run largo
make smoke

Nada de magic — cada target es un alias de python scripts/X.py --config config/base.yaml
